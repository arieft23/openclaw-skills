### The Problem

BBCA has **extremely strong broker data**:
- Perfect broker score (100)
- Very high buy ratio (81%)
- Strong 5-day cluster
- Broad broker participation (40 unique brokers)

However, because `insider_score = 0` (no insider activity), the composite score is heavily penalized, resulting in:
- Final Signal: **ACCUMULATION** (not HIGH_CONVICTION)
- Composite Score: **22/100** (very low)
- Signal Quality: **WEAK**

This doesn't reflect the genuine strength of broker accumulation.

## Root Cause Analysis

The current scoring algorithm appears to heavily weight insider data. When insider = null, even excellent broker data cannot push the signal to HIGH_CONVICTION.

### Possible Issues in `enrich_unified.py`:

1. **Penalty for single-source:** Broker-only signals may receive excessive penalty
2. **Weighting formula:** Insider may be weighted 70%+ of final score
3. **Minimum threshold:** HIGH_CONVICTION may require BOTH sources to have minimum scores

---

## Recommendations

### Option 1: Broker-Strong Override
Add logic to upgrade ACCUMULATION → HIGH_CONVICTION when:
- Broker score ≥ 80
- Broker buy ratio ≥ 75%
- Broker cluster ≥ 5 days
- No negative signals from other sources

### Option 2: Adjust Weighting
Reduce insider weight from ~70% to ~50%, giving more weight to broker data:
```
composite_score = (insider_score * 0.5) + (broker_score * 0.5)
```

### Option 3: Add Broker-Only Signal Type
Create new signal type `BROKER_STRONG` for cases with:
- Strong broker accumulation
- No insider data (not negative, just absent)
- This is a valid market scenario

### Option 4: Minimum Thresholds
Instead of pure weighted average, use:
- If insider_score > 0: weighted average
- If insider_score == 0 AND broker_score > 80: use broker_score directly (with discount)
